Privilege / consultant workflow

· Work & Productivity

How a consultant uses Privilege on a real engagement — without pasting the client file into ChatGPT, Codex, or Claude. Raw stays local; the agent only triggers preflight on vaulted IDs.

01

Client sends a document

Contract, operating notes, diligence pack — the kind you must not paste into a cloud chat.

ON YOUR MACHINE
02

Import into Privilege

Drop PDF / DOCX / text into the local viewer or CLI. Extraction runs on-device into the vault.

LOCAL VAULT
03

Set what’s confidential

Protected names + plain-English rules. Preview shows what OpenAI would see as policy — no real names.

ENGAGEMENT POLICY
04

Ask your question

“Summarize depot costs.” Privilege sanitizes, then GPT-5.6 attacks the candidate plus prior disclosures.

PREFLIGHT → GPT-5.6
05

Decision + receipt

Allow, Transform, or Block — with an inspectable receipt of inferences and matched rules.

WRITTEN LOCALLY
06

Use the answer

On Allow, analysis runs on sanitized text; real names are restored only on your laptop. Keep asking; the ledger accumulates.

RESTORED LOCALLY
↓ from step 05 — what happens next

If Transform or Block TRANSFORM BLOCK

The send is stopped or rewritten. You tighten the ask or the policy and try again — you do not “hope the redaction held.”

  • Receipt shows what GPT-5.6 inferred
  • Mosaic risk: turn 4 can fail after 1–3 passed

If Allow ALLOW

GPT-5.6 does the real work on the sanitized payload. You get a restored answer for the client write-up — names never crossed the wire.

Optional · agent path

Mount Privilege via MCP in Codex / Cursor / Claude. Import first in the app; the agent only calls preflight / analyze on vault IDs — no raw-text tool.

Productivity loop: import once → ask many times → cumulative preflight guards the engagement · never hand the raw file to the cloud agent
